---
title: "Update Resource Fields"
api: "POST https://artemisarbackend.com:8080/ext/update_resource_fields"
description: "This endpoint updates fields for a specified resource."
---

### Body

<ParamField body="api_key" type="string" required>
  This is the API key for your account. You can obtain this from the "Secrets" tab of your "Settings" page.
</ParamField>

<ParamField body="subproject_id" type="string" required>
  This is the id of the subproject the resource resides in. You can obtain this ID from the URL while in the subproject.
</ParamField>

<ParamField body="resource_id" type="string" required>
  This is the id of the resource to be updated. You can obtain this ID by visiting the resource and looking at the table on the first tab of the resource modal.
</ParamField>

<ParamField body="version" type="string" required>
  This is the version of the resource to be updated. This is indicated in the "Version Dropdown" while viewing the resource. The default version is "Version 1".
</ParamField>

<ParamField body="fields" type="[Field]" required>
  An array of fields to be logged under the "Resource Fields" table. The structure of each Field item is detailed below.
   <Expandable title="Field Structure">
    <ResponseField name="name" type="string" required>
      The name of the field
    </ResponseField>
    <ResponseField name="type" type="string" optional>
        The updated type for this field. Acceptable types include: text, number, boolean
    </ResponseField>
     <ResponseField name="description" type="string" optional default="">
        The updated description for this field
    </ResponseField>
     <ResponseField name="default" type="type" optional default="">
        The updated default value for this field
    </ResponseField>
  </Expandable>
</ParamField>


### Response

<ResponseField name="status" type="number">
  Indicates whether the call was successful. 200 if successful, 400 if not.
</ResponseField>

<ResponseField name="message" type="string">
  Message saying 'Resource fields updated' if successful, an error message if not.
</ResponseField>

<RequestExample>

```bash Example Request
curl --location --request POST 'https://artemisarbackend.com:8080/ext/update_resource_fields' \
--header 'Content-Type: application/json' \
--header 'Authorization: Token <token>' \
--data-raw '{
    "api_key": "XXXXX",
    "subproject_id": "XXXXX",
    "resource_id": "XXXXX",
    "version": "Version 1",
    "fields" : [
      {
        "name": "Field 1",
        "type": "number",
        "description": "This is the new description",
        "default": "0"
      }
    ]
}'
```

</RequestExample>

<ResponseExample>

```json Response
{
  "status": 200,
  "message": "Resource fields updated"
}
```

</ResponseExample>
