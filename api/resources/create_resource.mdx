---
title: "Create Resource"
api: "POST https://artemisarbackend.com:8080/ext/create_resource"
description: "This endpoint creates a resource in a subproject."
---

### Body

<ParamField body="api_key" type="string" required>
  This is the API key for your account. You can obtain this from the "Secrets" tab of your "Settings" page.
</ParamField>

<ParamField body="subproject_id" type="string" required>
  This is the id of the subproject the resource will be created in. You can obtain this ID from the URL while in the subproject.
</ParamField>

<ParamField body="name" type="string" required>
  This is the name of the resource to be created. If you would like to create the resource in a folder, use the following format: Folder/Resource or MainFolder/SubFolder/Resource
</ParamField>

<ParamField body="type" type="string" required>
  This is the type of the resource to be created. Make sure this type exists in your organization.
</ParamField>

<ParamField body="description" type="string" optional default="">
  This is the description of the resource to be created.
</ParamField>

<ParamField body="content_notes" type="string" optional default="">
  Notes on the content of the resource to be created.
</ParamField>

<ParamField body="content" type="[Content]" optional>
  An array of content items to be logged under the "Resource Content" table. The structure of each Content item is detailed below.
   <Expandable title="Content Structure">
    <ResponseField name="name" type="string" required>
      The name of the content
    </ResponseField>
    <ResponseField name="version" type="string" optional>
        The version of the content
    </ResponseField>
     <ResponseField name="location" type="string" optional>
        The location of the content
    </ResponseField>
     <ResponseField name="description" type="string" optional>
        The description of the content
    </ResponseField>
  </Expandable>
</ParamField>

<ParamField body="fields" type="[Field]" optional>
  An array of fields to be logged under the "Resource Fields" table. The structure of each Field item is detailed below.
   <Expandable title="Field Structure">
    <ResponseField name="name" type="string" required>
      The name of the field
    </ResponseField>
    <ResponseField name="type" type="string" required>
        The type of the field. Acceptable types include: text, number, boolean
    </ResponseField>
     <ResponseField name="description" type="string" optional default="">
        The description of the field
    </ResponseField>
     <ResponseField name="default" type="type" optional default="">
        The default value of the field
    </ResponseField>
  </Expandable>
</ParamField>


### Response

<ResponseField name="status" type="number">
  Indicates whether the call was successful. 200 if successful, 400 if not.
</ResponseField>

<ResponseField name="message" type="string">
  Message containing the id of the resource created if successful, an error message if not.
</ResponseField>
